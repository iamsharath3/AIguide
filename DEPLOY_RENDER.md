# Deploying HACK2SKILL to Render

This guide explains how to deploy your **HACK2SKILL** application to [Render](https://render.com) using the created `render.yaml` Blueprint.

## Prerequisites

1.  A **GitHub** account.
2.  A **Render** account.
3.  Your project pushed to a GitHub repository.

## Steps

### 1. Push Code to GitHub

Ensure your latest code, including the new `render.yaml` file, is pushed to your GitHub repository.

```bash
git add .
git commit -m "Add Render deployment configuration"
git push origin main
```

### 2. Create New Blueprint on Render

1.  Log in to the [Render Dashboard](https://dashboard.render.com/).
2.  Click on the **New +** button and select **Blueprint**.
3.  Connect your GitHub account if you haven't already.
4.  Select the **HACK2SKILL** repository from the list.
5.  Render will automatically detect the `render.yaml` file.
6.  You will see the resources to be created:
    *   `hack2skill-app` (Web Service)
    *   `hack2skill-db` (PostgreSQL Database)

### 3. Configure Environment Variables

During the setup (or after creation in the "Environment" tab of the Web Service), you must provide the **GEMINI_API_KEY**.

*   `DATABASE_URL`: Automatically verified and linked by Render (Magic!).
*   `JWT_SECRET`: Auto-generated by Render.
*   `NODE_ENV`: Set to `production` automatically.
*   **`GEMINI_API_KEY`**: Enter your actual Gemini API Key here.

### 4. Deploy

Click **Apply** or **Create Blueprint**. Render will:

1.  Provision the PostgreSQL database.
2.  Build your Node.js backend.
3.  Run the build script (`npm install && npm run build`), which also installs client dependencies and builds the React app.
4.  Start the server using `node server/server.js`.

## Troubleshooting

-   **Logs**: Check the "Logs" tab in the Render Dashboard if the deploy fails.
-   **Database Connection**: Ensure the `DATABASE_URL` matches the internal connection string of the database (Render handles this automatically with Blueprints).
-   **Build Failures**: Verify that `npm run build` works locally (it should build the client into `client/dist`).
